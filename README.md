# project_pre-scheduler
5iveの案件管理システムを作りたい。

## 目的
1. 現在の実装のアサイン状況を入力し、全体像を把握。
2. 新しい案件を入れる余地を素早く見つけれるようにしたい。
3. 実装チームにとってわかりやすく。

## 技術
| 項目 | 情報 |
|---|---|
| Framework | Next.js |
| DB | Turso（外部DB） |
| Deploy | Vercel |

個人レベルの範囲で使う分には基本無料。

## コマンド

| 項目 | 情報 |
|---|---|
| npm run dev | 開発用DB（Turso）と接続して起動 |
| npm run preview | 本番用DB（Turso）と接続して起動 |

※注意
- `npm run dev` で起動している場合は開発用DBに接続しているため、自由に操作してよい。
- `npm run preview` で起動している場合は本番用DBに接続しているため、操作画面からの変更・保存は行わないこと。
  特に「設定」モーダルの「保存する」ボタンはカスタム休日を全削除・再登録するため、誤操作に注意。
